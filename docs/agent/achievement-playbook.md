# Agent Achievement Playbook

This file covers run setup, achievement targeting, shop priorities, and achievement-specific tactical exceptions. Use it before starting a run, choosing islands/missions, accepting objective loss, or handling named achievements.

See also `data/ref_achievement_strategies.md`, `data/achievements_detailed.json`, `TODO.md`, and the achievement retrospectives under `docs/`.

## Achievement Context

Progress persists in session. Local checklist currently tracks 54/70 Steam-cache confirmed after **Chronophobia** unlocked on 2026-06-14, with 16 remaining across 4 difficulty tiers (Green >40%, Yellow 20–40%, Orange 10–20%, Red <10%).

- Squad-specific achievements need the matching squad — check `data/ref_achievement_strategies.md`.
- Cumulative achievements (reputation, civilians, pilot reuse) accrue across runs.
- Achievement strategies bias solver scoring via `EvalWeights` in `src/solver/evaluate.py`.

## Detailed Achievement Rules

38. **Weapon upgrades that change behavior must be explicit in solving and execution.** The Lua bridge reports the pawn's base SkillList, so powered upgrades that change semantics need save-file overlays plus Rust `WId` support, and bridge execution must fire the save-file effective weapon id for that slot. `Ranged_Artillerymech_A` is Artemis Artillery with direct Grid Building damage set to zero; push/bump collision damage is still physical and can still hurt buildings. `Science_Swap_A/B/AB` are Teleporter range upgrades (2/3/4 tiles); without those IDs the solver cannot find Quantum Entanglement lines, and without save-effective bridge firing `Pawn:FireWeapon` can ACK `Science_Swap` while silently no-oping a 4-range shot. If a save loadout contains a new `_A` / `_B` / `_AB` weapon and the overlay drops it or the bridge ACK shows the base id during an upgraded shot, add the upgraded ID before trusting the solve.

42. **Some weapon-specific kill pushes still corpse-bump live blockers.** `Ranged_Rocket` center damage and forward push are simultaneous: if the center target dies, its corpse still resolves the Rocket center push. `Prime_Punchmech` / Titan Fist and Dash Punch, `Prime_Flamethrower`, and `Prime_Leap` / Hydraulic Legs also resolve killed-target corpse pushes into live blockers. A dead pushable center/adjacent target can bump a live blocker; a dead non-pushable target can still bump static blockers such as buildings/mountains before disappearing when that weapon's policy allows it. Do not generalize this to every push: Cluster Artillery outer corpse absorption still has a regression test where a dead train corpse does not damage a live adjacent unit. Regression anchors: Easy run `20260506_114649_974`, Storage Vaults turn 1 Pulse mech death and turn 4 Jelly Armor building bump, simulator v55; Easy Flame Behemoths Perfect Strategy run `20260517_175633_388`, Mission_Teleporter turn 4 TeleMech death from a killed Scorpion corpse, simulator v144; Hazardous Mechs Healing run `20260522_154935_555`, Corporate HQ turn 3, where a killed D4 Leaper corpse-bumped UnstableTank on D5, simulator v190; Random Squad Change the Odds run `20260527_152006_916`, Corporate HQ turn 1, where plain Titan Fist killed BlobB on B6 and its corpse bumped Boulder Mech at A6, simulator v234.
42a. **Disabled mech wrecks block pushes.** A disabled/dead mech wreck (`board.wreck_at`) is a static blocker for pushed units. If a live unit is pushed into a wreck, the pushed unit takes 1 bump damage and stays in place; the wreck remains inert. Regression anchor: Hazardous Mechs Healing run `20260522_154935_555`, Corporate HQ turn 4, where `Prime_Leap` pushed an acid Beetle Leader from C6 toward disabled Nano's wreck on C7; live applied acid-doubled Leap damage plus 1 bump and left the boss on C6, simulator v192.

44. **Time pod recovery UI beats the crossed objective line.** Mission objective rows can look failed after a pod is collected, while the reward flow still shows `Pod Recovered` and the save has `podReward`. Treat the reward screen / save reward state as authoritative. In fallback parsing, numeric save pod state `1` means live/uncollected; recovered state `3` must not be treated as a live board pod.

83. **Focused Rust unit tests need the cargo test harness flags.** This PyO3 crate defaults to `extension-module`; direct local unit-test runs must use one filter plus `--no-default-features`, and on macOS may need `DYLD_FRAMEWORK_PATH=/Applications/Xcode.app/Contents/Developer/Library/Frameworks` so the test binary finds Python3.framework (for example `CARGO_INCREMENTAL=0 DYLD_FRAMEWORK_PATH=/Applications/Xcode.app/Contents/Developer/Library/Frameworks cargo test --no-default-features sand`). Passing multiple test-name filters, omitting `--no-default-features`, or omitting the framework path can fail before the useful simulator test signal. Rust unit tests that call Python APIs (`Python::with_gil`, pyfunctions, etc.) also need `pyo3::prepare_freethreaded_python()` before acquiring the GIL because this crate does not enable PyO3 auto-initialize in the test harness. Regression anchor: Rusting Hulks There is No Try run `20260518_215903_654`, Pinnacle Central Processor v149 patch validation, where direct `cargo test test_a test_b` first rejected the second filter and then direct `cargo test test_a` hit PyO3 undefined symbols before the corrected `--no-default-features` command passed.

92. **Installed reactor cores are not temporary cross-mech rentals.** A core installed into a mech's reactor stays on that mech; you can toggle powered upgrades within that mech, but do not assume the core can be pulled back and reassigned to another chassis later. When an achievement needs a specific upgraded weapon (for example Titan Fist Dash on Combat Mech), keep newly bought cores uninstalled or install them directly into the required mech. Regression anchor: Easy Rift Walkers run `20260510_213059_819`, after Detritus Corporate HQ, a "temporary" Artillery Buildings Immune core could not be moved to Combat Mech for Ramming Speed.

102. **Timeline Lost pilot carry-forward is click/select UI, not drag setup.** On the post-loss Time Traveler screen, choose the carry-forward pilot by clicking/selecting the visible pilot/card controls and then continue to setup; do not drag pilots on this screen unless the game shows a specific drag target. Once the hangar/setup screen is visible, verify carried pilot, squad, difficulty, Advanced Edition, and target setup before pressing Start. Regression anchor: Easy Rift Walkers run `20260510_213059_819` post-loss cleanup.

129. **Hard Rusting Hulks should prefer a third island unless Hive-ready.** For Hard Victory with Rusting Hulks, a 2-island Volcanic Hive attempt is acceptable only when grid is full or near-full *and* the kit is final-ready: Jet has range/control enough to cover stacked threats, Rocket has meaningful damage/range, Pulse has powered Repulse, and pilots/mech HP are stable. `new_run` stamps Hard Rusting Hulks attempts with `Hard Victory` / `hard_victory` metadata so Hard-only gates fire. `6/7` grid alone is not readiness; if Jet and Rocket are still base kit, take island 3 for cores/grid/control before the Hive.

132. **Mission_Wind requires live WindDir plus simulator v148+.** Wind Storm rows are push lanes, not direct damage tiles: the live mission script applies `SpaceDamage(... iPush = WindDir)` across marked rows before enemy attacks, so mechs/enemies can bump buildings or re-aim from pushed tiles. The bridge must export `environment_wind_dir` (reader fallback reads `LiveEnvironment.WindDir` from saveData) and Rust must apply `board.env_wind` before Vek attacks. If a Mission_Wind solve input has wind rows but lacks `environment_wind_dir`, do not trust clean/dirty candidates that leave a mech, Vek, or building-impacting push chain on a wind row; restart with the installed modloader or stop for investigation. Regression anchors: Hard Rusting Hulks run `20260513_213619_103`, Mission_Wind turn 1, rank-114 line predicted grid `5/7` but live wind left grid `3/7`; Easy Rusting Hulks Perfect Strategy run `20260518_184029_155`, Mission_Wind turn 2, RocketMech C3 was wind-bumped into the Old Earth Bar at B3. Fixed in simulator v148.

139. **Mission_Terraform grassland is a custom-sprite objective, not terrain id.** The "Terraform the grassland back to desert" bonus tracks `ground_grass.png` custom tiles in the save, while the bridge terrain id can still say ordinary `ground`, `water`, or `sand`. At a fresh player-turn boundary, overlay active-region custom grass from saveData into `tile.grass`; do not overlay it mid-turn because saveData is stale after bridge actions. Rust must score remaining grass on `Mission_Terraform`, and `Terraformer_Attack` must clear the grass flag when it converts a non-mountain tile to sand. Regression anchor: Hard Rusting Hulks run `20260513_230944_542`, Sierra Quadrant reward screen showed `Transform the grassland back to desert (Failed)` even though grid, pod, and Terraformer protection were clean.

179. **Perfect Strategy farming should avoid spawn-block objectives until objective scoring is explicit.** "Block Vek Spawning N times" can look like a safe 1-star mission, but the current live loop optimizes survival/threats and may finish clean combat while under-blocking spawns. Do not pick spawn-block objectives for Perfect Island grinding unless the solver output explicitly tracks and completes the required count before mission end, or a dedicated objective-weight fix has landed. Regression anchor: Flame Behemoths Easy run `20260517_183219_983`, R.S.T. `Mission_Survive` / Turbine Sinks, "Block Vek Spawning 3 times" ended at 1/3 and failed despite Coal Plant survival.

185. **`recommend_mission` has no achievement/tag flags.** The CLI accepts only `--profile` and optional `--island-map-json`; do not pass `--tags` or `--achieve` while live-playing the map. For Perfect Strategy, use the default scorer plus visual objective verification, or pass an explicit offline map JSON if the bridge is not on the corporation map screen.

189. **Mech-damage-cap objectives are hard objective gates.** A visible objective like "End battle with less than 4 Mech Damage" fails as soon as the counter reaches 4; normal `mech_hp_loss` warnings are not enough. When active `BonusObjs` includes id `4`, carry a `mech_damage_objective_limit=4` summary field and make `mech_damage_objective_failed` non-overridable, even if grid/buildings are clean. Regression anchor: Perfect Strategy run `20260517_200713_348`, Detritus `Mission_Survive` / Reclamation Zone turn 3, a clean building plan moved Flame Mech onto fire, raised objective progress from 3 to 4, and killed the perfect island.

193. **Perfect Strategy avoids spawn-block objectives unless progress is hard-gated.** Missions with bonus text like "Block Vek Spawning 3 times" are not low-risk for Perfect Island farming unless the mission picker and turn solver explicitly treat the remaining block count as a hard objective before the final End Turn. If the first mission slate offers a spawn-block mission and a fragile-defense alternative, prefer restarting/choosing another island rather than trusting generic clean grid/building safety. During live play, if the objective counter cannot still reach its target from visible spawn tiles and available blockers, treat the Perfect Island as failed and abandon/restart after the mission. Regression anchor: Perfect Strategy run `20260517_222204_041`, Pinnacle `Mission_Shields` / Robotics Repair, where `auto_turn` preserved grid/objective buildings but ended with `Block Vek Spawning 3 times (Failed)` at 1/3.

197. **Frozen-building thaw objectives are counter objectives, not building safety.** `Mission_FreezeBldg` / "Break 5 buildings out of the ice" requires five objective buildings to be thawed before the final End Turn; preserving those buildings while frozen still fails the Perfect Island. During Perfect Strategy farming, require the solver payload to carry `freeze_building_target` and `freeze_building_tiles`, require visible/read progress to remain reachable from remaining active actions and turns, and stop before End Turn if it cannot reach 5. If progress is lagging by turn 3 or turn 4, restart/abandon rather than trusting generic no-grid-loss safety. Regression anchor: Flame Behemoths Easy run `20260517_231845_078`, Pinnacle `Mission_FreezeBldg` / Venting Field, where simulator v146 thaw mechanics worked but low objective weighting reached only 2/5 by final End Turn and Reset Turn was unavailable after bridge actions.

200. **Perfect Strategy avoids minimum-kill objectives unless kill progress is hard-gated.** Missions with bonus text like "Kill at least 5 Enemies" are unsafe for Perfect Island farming unless the solver safety summary proves `mission_kills_done + predicted_enemy_phase_kills >= mission_kill_target` before the final End Turn. Scoring partial progress is not enough; a clean grid/objective-building plan can still finish at 3/5 with all mechs DONE and Reset Turn unavailable after bridge actions. For Perfect Strategy, reject `BONUS_KILL_FIVE` previews when any safer non-counter mission exists. If forced into one, inspect the visible counter and bridge `mission_kills_done` before every End Turn; on the final turn, do not advance unless the remaining live enemies are actually lethal this enemy phase (for example 1 HP on Fire), not merely burning at high HP. Regression anchor: Flame Behemoths Easy Perfect Strategy run `20260518_085625_707`, R.S.T. `Mission_Survive` / The Haze turn 4, where the visible counter stayed 3/5 and two burning Scorpions were still 3 HP.

210. **`Mission_Tanks` v147 regression check.** The visible objective `Defend the Tanks (2/2 undamaged)` failed in run `20260518_105028_125` because pre-v147 safety had `protected_objective_units_alive=0` and Flame Shielding incorrectly protected non-mech allied tanks from fire. After any future tank/fire change, replay the old line with `Mission_Tanks` metadata injected and require it to classify as `DIRTY` with `protected_objective_unit_lost`; a clean result means the objective guard regressed. Keep avoiding Tanks missions for Perfect Island farming unless no safer alternative exists.

211. **Codex `spawn_agent` full-history forks cannot override type/model/reasoning.** When using `fork_context=true`, omit `agent_type`, `model`, and `reasoning_effort`; otherwise the app rejects the spawn before the agent starts. If a specialized explorer/worker type is needed, spawn without full-history fork and provide the relevant paths/context explicitly. Regression anchor: Flame Behemoths Easy Perfect Strategy run `20260518_105028_125` failure postmortem, where a forked explorer with explicit `agent_type`/`reasoning_effort` was rejected twice.

215. **Post-loss squad-screen Back exits to main menu.** After selecting a carry-forward pilot on the `Timeline Lost` flow, the large `Back` button on the squad/start screen returns to the main menu rather than the difficulty/Advanced Content setup panel. To re-open setup verification, use `New Game` from the main menu and proceed through the normal setup screen. Regression anchor: simulator v147 restart after `Mission_Tanks`, where clicking Back to find the difficulty panel returned to the title menu.

218. **`mission_end` has no `--perfect` flag.** Record mission outcomes with the supported positional/status options from `python3 game_loop.py mission_end -h`; put perfect/failed-objective details in `--notes` instead of inventing `--perfect`. Regression anchor: Flame Behemoths Perfect Strategy run `20260518_121221_583`, Archival Flats reward screen recording.

220. **Prime Flamethrower target-tile fire is flight-sensitive.** `Prime_Flamethrower` lights the struck tile before resolving its forward push. A grounded occupant catches that newly created tile fire even if the push moves it away; an effectively-flying occupant does not carry the new fire status unless the push leaves it on or lands it on a burning tile. Do not model this as either "all pushed targets catch fire" or "no pushed target catches fire." Regression anchors: Flame Behemoths `This is Fine` run `20260520_174936_811`, `Mission_ForestFire` turn 4 Mosquito stayed not-on-fire after being pushed off the lit tile (sim v158), then `Mission_Mines` turn 1 Bouncer did catch fire while being pushed off the lit tile because it was grounded (sim v159).

221. **Fire weapons do not create burning water tiles.** A Fire weapon can set a flying unit over Water on fire, but Water itself does not gain the burning tile flag. For `Prime_Flamethrower`, if a flying target over Water is blocked and remains on the struck tile, apply direct Fire to the unit while leaving the Water tile non-burning. Regression anchor: Flame Behemoths `This is Fine` run `20260520_174936_811`, `Mission_Mines` turn 2, where a blocked Mosquito over B4 water caught fire but the B4 water tile stayed non-burning; fixed in simulator v160.

220. **Verify Corporate HQ is truly active before skipping side missions.** `recommend_mission` can return `NO_ISLAND_MAP` on the visible Archive map after reward flow, so visual selection still matters. Click Corporate HQ only to inspect it first; if the preview says `No Vek Detected`, the HQ fight is not unlocked yet and another side mission is required. Then choose the least risky remaining side mission, preferring controllable counters over trains, fragile allies, or minimum-kill objectives. Regression anchor: Flame Behemoths Perfect Strategy run `20260518_121221_583`, after Retrospect Park, where Corporate HQ inspected as `No Vek Detected` and Preserved Farms showed `Use 3 Repair Platforms` plus `Kill at least 5 Enemies`.

221. **Distant Friends setup should avoid train missions when alternatives exist.** The hidden-pod hunt cares about reaching islands 2-4 with a live timeline, not about squeezing first-island reputation from brittle protected-unit maps. With Frozen Titans in particular, Archive `Mission_Train` can safety-block on an unavoidable protected train loss before the pod hunt even starts. If the first-island preview says `Defend the Train`, choose a non-train alternative when one can be selected safely, or restart the island choice rather than accepting protected-objective loss. Regression anchor: Frozen Titans Distant Friends run `20260525_201414_595`, Archive `Archival Flats` turn 2, where all 1000 widened candidates predicted `protected_objective_unit_lost`.

222. **Close stale subagents before spawning during live play.** Codex can hit an agent-thread limit even when a quick explorer would be useful. If `spawn_agent` fails with `agent thread limit reached`, close old completed/stale agents from the thread context before trying again, or continue locally if the decision is time-sensitive. Regression anchor: Flame Behemoths Perfect Strategy run `20260518_121221_583`, post-Antiquity mission-choice triage.

228. **Perfect Strategy avoids Mite counter missions unless progress is hard-gated.** `Mission_Holes` / "Knock Mites off the Mechs" is a counter objective; generic grid/building/pod safety does not prove it. During Perfect Island farming, reject Mite missions when any alternative exists unless the solver payload exposes `mites_remaining`/objective progress and the final-turn safety requires it to reach zero before End Turn. If forced into one, inspect the visible objective counter before every End Turn; on the final turn, do not advance while `Remaining` is nonzero, even if all threats, grid, Coal Plant, and Time Pod are safe. Regression anchor: Flame Behemoths Perfect Strategy run `20260518_121221_583`, R.S.T. `Mission_Holes` / Hardin Point, where final reward showed `Knock Mites off the Mechs (Failed)` with two mites remaining.

239. **Kill-N objectives are final-turn hard gates, not scoring hints.** `mission_kill_target` / `mission_kills_done` must be carried into board summaries and plan safety; on the final player turn, do not execute or End Turn unless the planned line reaches the target. If bridge actions already spent the turn and Reset Turn only shows a tooltip/no confirmation, treat the mission as likely non-perfect, log it, and restart the timeline rather than accepting a failed Perfect Island. Regression anchor: Flame Behemoths Perfect Strategy run `20260518_160125_397`, Pinnacle `Mission_SnowStorm` / Cold Storage turn 4, where a safety-WARN plan ended with `Kill at least 5 Enemies (Current: 4)`.

240. **Mission_Force mountain objectives are counter objectives.** `Mission_Force` / "Destroy 2 mountains" must expose `mission_mountain_target`, `mission_mountains_destroyed`, and the current solve-input mountain tiles so replay can count newly-rubbled mountains. On the final player turn, safety must hard-block and remain non-overridable if the projected counter is below target. During Perfect Strategy farming, do not trust a generic clean grid/building audit for this mission unless `mission_mountains_destroyed >= mission_mountain_target` is present in the safety summary or the visible row is already complete. Regression anchor: Rusting Hulks Easy Perfect Strategy run `20260518_184029_155`, R.S.T. Black Rock / `Mission_Force`, where early clean plans ignored the visible `Destroy 2 mountains (Current: 0)` counter.

244. **Burrower missing-after-damage diffs are known conservative board drift.** Burrowers can leave the visible board after being damaged/smoked and later return through normal enemy logic; if a per-action verify reports only `missing_in_actual` for a known `Burrower*` after a nonlethal transit/weapon hit, with no grid/objective/pod loss and a fresh replay otherwise clean, treat it as conservative drift for the current End Turn. Do not partial re-solve from the bridge snapshot that temporarily omits the Burrower, because that can erase a latent same-uid threat from the prediction; keep the conservative predicted board and rely on the next fresh bridge read after End Turn. Resolve the behavior-novelty queue item with an explicit reason if one was queued. Regression anchors: Rusting Hulks There is No Try run `20260518_215903_654`, Pinnacle Central Processor turn 2, Aerial Bombs damaged/smoked Alpha Burrower on F2 and live removed it while sim v149 still projected it at F2; Heat Sinkers Feed the Flame run `20260617_222359_646`, Detritus Reprocessing turn 2, where partial re-solve dropped a burrowed `Burrower1` uid 30 and falsely predicted no grid/mech damage.

245. **Block-spawn objectives are hard gates for no-failed-objective achievements.** `BONUS_BLOCK` / "Block Vek Spawning N times" is not covered by the generic grid/building/objective-building safety summary; a plan can be `CLEAN` while the visible counter remains short and later fails on the Region Secured screen. During There Is No Try / Perfect Island / any no-failed-objective push, avoid `BONUS_BLOCK` missions unless the counter is explicitly modeled and projected complete. If a `BONUS_BLOCK` mission is forced or accidentally launched, treat every End Turn as unsafe unless the visible/save counter can still reach the target; if it cannot, abandon/restart the achievement line instead of counting the region as valid. Regression anchor: Rusting Hulks There is No Try run `20260518_215903_654`, Pinnacle `Mission_Stasis` / Shielded Vaults, where turn plans stayed grid/objective clean but the final screen showed `Block Vek Spawning 3 times (Failed)`.

247. **Minor Vek kills are not mission.KilledVek.** Totems, spiderlings, blobs/minis, eggs, and other `minor=true` enemy pawns are still enemies for tactics and normal kill scoring, but they do not advance `mission.KilledVek` counters such as `BONUS_KILL_FIVE` / "Kill at least 5 Enemies". Mission-kill objective accounting must use a separate objective-counted kill total, not generic `enemies_killed`. During no-failed-objective pushes, if the final-turn kill counter depends on a minor enemy, treat the plan as short unless another non-minor kill is guaranteed. Regression anchor: Rusting Hulks There Is No Try run `20260519_133059_179`, Archive `Mission_Repair` / Renovation Complex turn 4, where the solver projected 2 + Firefly + Firefly + Totem = 5 but Region Secured showed `Kill at least 5 Enemies (Failed)` at 4/5.

249. **Vek Mites make Repair non-noop even at full HP.** Repair clears `bInfected` / Vek Mites, so action enumeration and auto-turn no-op guards must allow a full-health infected mech to repair. The bridge's direct `REPAIR` command must call `SetInfected(false)` in addition to clearing fire/acid/frozen; otherwise the solver can believe a final-turn mite objective is solved while live `Board:GetInfectedCount()` still fails it. Regression anchor: Rusting Hulks There Is No Try run `20260519_154655_297`, Detritus `Mission_Belt` / Downtown turn 4, where PulseMech at G5 was full HP, infected, and direct repair healed/consumed the action without clearing the remaining mite.

250. **Stale non-egg web ownership must be repaired before solving.** `web_probes.IsGrappled=true` / `web=true` is authoritative, but `web_source_uid` from the bridge can lag behind enemy retargeting. If a webbed unit has an alive queued web source currently targeting its tile, prefer that source over a stale non-egg `web_source_uid`; otherwise moving or smoking the wrong stale source can falsely clear the web in simulation. Regression anchor: Rusting Hulks There Is No Try run `20260519_154655_297`, Archive `Mission_Tides` / Archival Flats turn 3, PulseMech C3 stayed webbed after Rocket moved Scorpion uid 776 because the true queued source was Scorpion uid 799.

251. **Firefly Leader attacks are paired projectiles.** `FireflyAtkB` / Burning Thorax queues a projectile toward the marked target and a second projectile in the exact opposite direction from the leader's current tile. Simulating only the forward shot can miss ordinary or objective building loss behind the leader; the backward shot also uses `GetProjectileEnd` and is skipped only when it would immediately hit the leader's own tile off-board. Regression anchor: Rusting Hulks There Is No Try run `20260519_154655_297`, Archive `Mission_FireflyBoss` / Corporate HQ turn 2, where the leader's backward shot destroyed the B6 2-HP building while the solver predicted no grid loss.

252. **Mission_Dam's dam is a destroy-objective unit, not optional scenery.** `Mission_Dam` must inject `Dam_Pawn` into `destroy_objective_unit_types`; otherwise safety can finish a clean grid/building line while the reward screen says `Destroy the Dam (Failed)`. If the final-turn solve summary has `mission_id=Mission_Dam` and `destroy_objective_units_alive=0` while the visible objective still says `Destroy the Dam`, stop before End Turn and fix the metadata/replay path. Regression anchor: Flame Behemoths Quantum/This Is Fine run `20260519_224158_398`, Archive Secondary Archives, where the final plan protected grid/pod but never destroyed `Dam_Pawn`.

253. **Science_Swap / Teleporter is cardinal-line only.** Upgraded Teleporter range expands along the four cardinal directions; it does not make a Manhattan-radius diamond or square target area. A bridge ACK such as `FireWeapon[1](Science_Swap_AB)` on a diagonal target can spend the action without moving anything, so treat any diagonal Teleporter solve as a solver enumeration/simulator bug and patch before continuing. Regression anchor: Flame Behemoths Quantum/This Is Fine run `20260519_224158_398`, Archive `Mission_Holes` turn 2, where TeleMech D7 tried diagonal C6 and live performed no swap despite the upgraded weapon ACK.

255. **FIRE weapon status consumes Sand/Forest into burning Ground.** When a weapon applies direct FIRE status to a Sand or Forest tile, live verification reports the terrain as Ground with fire, not the original terrain with a fire flag. If a Flamethrower / Ignite verify diff is only `terrain sand->ground` or `forest->ground` while fire/grid/objective/threat state matches, patch/rebuild the status terrain path and then continue; do not classify it as a tactical failure. Regression anchor: Flame Behemoths Quantum/This Is Fine run `20260519_224158_398`, Archive `Mission_Holes` turn 3, where FlameMech fired at D5 sand and live produced burning Ground.

256. **Pinnacle FACTION_BOTS do not receive Psion auras.** Snowbots (`Snowtank*`, `Snowart*`, `Snowlaser*`, `Snowmine*`) and Bot Leader variants are sentient weapons, not Vek. Soldier/Boss/Armor/Boost/Fire/Regen/Blast/Spider Psion aura logic must exclude them even though they are `TEAM_ENEMY`; otherwise killing `Jelly_Health1` can falsely subtract 1 HP and erase bots that live long enough to attack. Regression anchor: Flame Behemoths Quantum/This Is Fine run `20260520_134643_900`, Pinnacle `Mission_Factory` turn 2, where a Cannon-Bot survived the Soldier Psion death and destroyed a non-objective building while sim v156 predicted all enemies dead.

267. **Boosted Flame Thrower damage is conditional, not base damage.** Kai/Morgan Boost adds +1 weapon damage to `Prime_Flamethrower` only when the "Damage units already on Fire" branch is active. A boosted Flame Thrower does **3** damage to an already-burning target (FireDamage 2 + Boost 1), but still does **0** direct damage to a non-burning target. Powered range IDs `Prime_Flamethrower_A/B/AB` must be overlaid from the save and modeled as range/path sizes 2/2/3. Regression anchor: Flame Behemoths This is Fine run `20260520_174936_811`, R.S.T. `Mission_Force` turn 4, where boosted Kai killed a burning 3-HP `Starfish2` at E5 and pre-v161 Rust predicted it survived at 1 HP.

269. **Duplicate mission-unit segments share web state.** Multi-tile units such as `Train_Pawn` can appear as duplicate board entries with the same uid. When a web source moves/dies and the engine breaks the grapple, every segment with that logical uid clears `web`, even if only the primary segment carried `web_source_uid` in the bridge payload. Regression anchor: Flame Behemoths This is Fine run `20260520_174936_811`, R.S.T. `Mission_Train` turn 3, where swapping the webbing `Spider2` cleared web from both train segments and pre-v163 Rust left the extra segment webbed.

271. **Science Swap breaks web on moved targets.** `Science_Swap` / Teleporter relocates both the mech and the target through the engine's move-classed teleport path. If either relocated unit was webbed, the actual tile change clears that unit's own web; a stale `web_source_uid` from a `WebbEgg1` does not persist after the target is swapped away from adjacency. Regression anchor: Flame Behemoths This is Fine run `20260520_174936_811`, R.S.T. `Mission_Crack` turn 3, where TeleMech swapped webbed `Moth2` from E3 to C3 and live cleared `status.web` while pre-v165 Rust kept it webbed.
