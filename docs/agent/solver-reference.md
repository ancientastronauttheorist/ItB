# Agent Solver Reference

This file carries architecture, build/test discipline, core game mechanics, and the simulator/bridge regression casebook. When editing Rust, bridge Lua, parser/model code, or weapon semantics, read the relevant sections here first.

See also `docs/reference.md`, `docs/lua_bridge_architecture.md`, `docs/env_hazards_by_island.md`, and `data/ref_game_mechanics.md`.

## Architecture

**Five layers:**

- **Layer 0 — Game loop:** `game_loop.py` CLI + `src/loop/` (session, logger, commands). The agent is the control loop; Python commands are stateless.
- **Layer 1 — State extraction:** `src/bridge/` (primary) and `src/capture/save_parser.py` (fallback). The bridge delivers richer data: targeted tiles, per-enemy attack data, environment hazards with kill flag, deployment zones, objective buildings, attack order.
- **Layer 2 — Game state:** `src/model/` (Board, Unit, WeaponDef). Single source of truth for solver.
- **Layer 3 — Solver:** `rust_solver/` (the only solver AND the only simulator, pyo3 extension `itb_solver` with entry points `solve` / `solve_top_k` / `solve_beam` / `score_plan` / `project_plan` / `replay_solution`). Search is constraint-based threat response + bounded search. `EvalWeights` tune scoring without changing search logic. On the Python side: `src/solver/solver.py` is now a 121-line wrapper holding `MechAction` / `Solution` dataclasses and `replay_solution()` (a thin shim around `itb_solver.replay_solution` that adds Python's `evaluate_breakdown` for audit-only `score_breakdown`). `src/solver/evaluate.py` owns weight/threat/psion machinery for that breakdown — never touched by `solve()`. **`src/solver/simulate.py` is gone** (deleted in the simulate.py-removal PR series); every diagnose-loop fix targets `rust_solver/src/*.rs`, validator + agent prompt enforce `target_language=rust`.
- **Layer 4 — Strategist:** `src/strategy/` picks `EvalWeights` per achievement target; manages squad/island/shop choices.

**Rebuild the solver** after editing any `rust_solver/src/*.rs`:
```bash
cd rust_solver && maturin build --release && \
  pip3 install --user --force-reinstall target/wheels/itb_solver-0.1.0-cp39-cp39-macosx_11_0_arm64.whl
```
If `maturin` is not on `PATH`, use `python3 -m maturin build --release`
from `rust_solver/` with the same `pip3 install` command.

If `cargo test` hits a rustc incremental-compilation ICE, rerun with
`CARGO_INCREMENTAL=0` before treating it as a code/test failure. The regression
script already sets this guard.

Focused Rust unit tests in this PyO3 crate must use `cargo test --no-default-features`
so they link against Python instead of the extension-module defaults; otherwise
macOS reports unresolved `_Py*` symbols before any simulator assertion runs.

If `cargo test` or `maturin build --release` appears idle while Rust is reading
generated Cargo fingerprint files under `rust_solver/target/{debug,release}/.fingerprint`,
stop the stuck build, remove only that generated `.fingerprint` directory for the
affected profile, and rerun the same command. This is build-cache cleanup, not a
source revert. On macOS pyo3 test binaries may also need:
`DYLD_FRAMEWORK_PATH="/Applications/Xcode.app/Contents/Developer/Library/Frameworks:/Library/Developer/CommandLineTools/Library/Frameworks"`.

## Core Game Rules (Solver-Critical)

The solver enforces these; use them when reviewing solver output or writing tests.

**Terrain kills:** Water and Chasm kill non-flying ground units. Lava kills like water but sets flying units on Fire. Pushing enemies into these is a primary kill method.

**Push:** 1 tile in a direction. If blocked by unit/mountain/edge, the pushed unit takes 1 bump damage instead. If blocked by a building, *both* pushed unit and building take 1 bump. Chain pushing (A pushed into B) doesn't move B — both take bump. Push/bump damage ignores Armor and ACID.

**Damage types:**
- Weapon damage: −1 from Armor, ×2 from ACID
- Push / bump / fire / blocking damage: ignores Armor and ACID
- Blocking damage fires when a Vek tries to emerge on an occupied tile

**Status effects:**
- **Fire:** 1 dmg/turn start. Removed by repair, water, or freezing.
- **Frozen:** invincible + immobilized. Any damage unfreezes (0 dmg dealt).
- **ACID:** doubles weapon damage. Persists until unit dies.
- **Smoke:** prevents attack AND repair. Cancels Vek attacks that are already
  on smoke when the queued enemy attack loop begins; smoke created by an
  earlier enemy attack in the same loop does not retroactively cancel later
  queued attacks.
- **Shield:** blocks one instance of damage + negative effects. Removed by direct damage.
- **Armor:** −1 weapon damage (floor 0). No effect on push/fire/bump.
- **Webbed:** can't move, can still attack. Breaks only when the unit actually changes tiles, or when the webber moves/dies. A blocked push/bump leaves the unit webbed. If multiple live queued web sources target the same unit, killing/pushing one source transfers ownership to another source rather than freeing the unit.

**Terrain HP:**
- **Mountain:** 2 HP (2=full, 1=damaged, 0=rubble/walkable). Any weapon damage reduces by 1.
- **Building:** 1 HP (objective buildings 2). Any damage destroys all on the tile. Reduces grid power by old HP.
- **Ice:** intact → cracked → water (non-flying drowns).

**Repair platforms:** Mission_Repair places `Item_Repair_Mine` tiles. Any live
unit that lands on one triggers the item, heals up to max HP, consumes the
platform, but only a player mech increments `repair_platforms_used` toward the
3-platform objective. If the unit is already at or above max HP when it lands,
the platform still consumes without adding HP; it counts only when that unit is
a player mech. Raw progress can exceed the objective target (for example `4/3`);
clamp only in UI scoring/presentation. It is not the mech Repair action; do not
assume it clears Fire/ACID/Frozen unless a capture proves the engine does so.

**Repair:** any mech can repair instead of attacking. +1 HP, clears Fire and ACID. Can't repair on smoke.

Extended rules: `data/ref_game_mechanics.md`.

## Detailed Solver And Simulator Rules

1. Always verify state after each mech action (the bridge does this in `auto_turn` automatically). Save file only updates at turn boundaries; bridge updates per sub-action.

6. Priority order: buildings > threats > kills > spawns.

8. Never move onto ACID tiles voluntarily — doubles damage, disables armor.

9. **Self-improvement:** every process error triggers an immediate AGENTS.md update with a guard or fix. Every mistake makes the process permanently better.

12. Use all mech actions every turn — even suboptimal moves beat skipping.

13. The solver handles environment warnings via effect-specific channels. `environment_danger_v2` entries are `[x, y, damage, kill_int, flying_immune]`; `kill_int=1` is lethal (Air Strike, Lightning, Cataclysm→chasm, Seismic→chasm, Tidal→water; bypasses shield/frozen/armor/ACID for grounded units). A `kill_int=0` marker may route to a non-lethal effect such as Wind or Terratide smoke rather than direct damage. Ice Storm uses `environment_freeze`. Env ticks normally fire BEFORE Vek attacks in enemy phase. `Mission_Tides` is the exception: queued Vek attacks land before the wave advances, and flying units on the wave tile take 1 damage instead of being fully spared.

14. On crash/timeout recovery, always start with `cmd_read` + `cmd_solve`. Never resume a previous solution — the board may have changed.

15. Save file updates only at turn boundaries. Bridge does not have this limit — `verify_action` reads fresh bridge state.

29. **Final-island save phase anomaly.** During `Mission_Final` and `Mission_Final_Cave`, `saveData.lua` can report `mission_ending` while the Lua bridge and visible screen are still in live combat. If `cmd_read` / `auto_turn` says `mission_ending` but the screen shows active final combat, trust the fresh bridge when `mission_id` is `Mission_Final` or `Mission_Final_Cave` and at least one mech is active. Conversely, if `auto_turn` waits out with `active_mechs=0`, it now returns `status="TERMINAL_OR_MISSION_END"` and clears the stale active solution; check the visible screen for Victory / Region Secured / rewards / defeat before issuing more combat commands.

34. **Projectile-grapple attacks are line attacks, not melee.** Burnbug/Gastropod proboscis weapons (`BurnbugAtk1`, `BurnbugAtk2`, `BurnbugAtkB`, `GastropodAtk1`, `GastropodAtk2`) trace from the attacker's current tile along the original queued direction until the first projectile blocker, then damage that blocker and grapple: hit pawns are pulled toward the attacker, objects pull the attacker toward the object. A vacated first target tile does NOT cancel the shot. If an unexpected grid loss follows a moved Burnbug/Gastropod, first check for a building farther down that line. Regression anchor: Normal run `20260504_210332_088`, mission 1 turn 1, F7 `(1,2)` building loss fixed in simulator v44.

35. **Trust engine terrain ids over stale terrain names.** Shipped map files and Lua constants have terrain id `5` = Ice, not Lava. Older bridge builds mislabeled id 5 as `"lava"`, causing false terrain-death predictions on Pinnacle ice and the Coal Plant loss chain in Normal run `20260504_210332_088`, mission 1 turn 2. Parser code must prefer `terrain_id` normalization when present; bridge Lua must derive `TERRAIN_NAMES` from engine globals (`TERRAIN_ICE`, `TERRAIN_LAVA`, etc.) instead of hand-maintaining numeric tables.

46. **Detritus Contraption barrages must target a non-source unit.** `Missiles_Shield` / `Missiles_OneDmg` inherit `Support_Missiles:GetTargetArea`, which excludes the firing Contraption tile even though the effect is global. Clicking the Contraption's own tile can spend the action without emitting any missiles. The solver must pick a live non-source unit as the representative target, and the simulator must exclude the source from Shield/Missile Barrage effects. Regression anchor: Easy run `20260506_114649_974`, Mission_Missiles turn 2 source-click no-op, simulator v59.

46a. **Detritus Contraption can act from smoke.** `Missile_Unit` has
`IgnoreSmoke=true`; smoke on the Contraption tile must not prune
`Missiles_Shield` / `Missiles_OneDmg` from action enumeration. If
`Mission_Missiles` reaches a reward screen short of four Contraption uses after
the unit sat in smoke, check the smoke legality gate before blaming target
selection. Regression anchor: Bombermechs No Survivors run
`20260630_181556_177`, Detritus `Mission_Missiles` turn 4, where a smoked
Contraption was skipped and Region Secured showed `Use the Detritus Contraption
four times (3/4)`; fixed in simulator v310.

48. **Normal/Alpha Spiders are pushable.** `Spider1` / `Spider2` throw eggs but are still normal pushable Vek; do not mark them like Burrowers or Psions. If Repulse or another push leaves a Spider unmoved in prediction while the engine moves it, check `src/model/pawn_stats.py` and solve payload `pushable` enrichment before changing Repulse logic. Regression anchor: Easy run `20260506_114649_974`, Mission_Missiles turn 4 `Science_Repulse_A` pushed Spider1 from E5 to E6; fixed in simulator v60.

49. **Teleporter move-then-attack targets use the post-swap tile.** `pawn:Move()` fires Mission_Teleporter pads before the bridge sends `ATTACK`; if the solver keeps a self-AoE target on the pre-swap pad, `pawn:FireWeapon()` can silently spend the action as an invalid target and produce no Repulse/shield effect. Enumerate attacks, smoke checks, and no-op filtering from the post-teleport board while keeping `move_to` as the pad tile. Invalid diagonal SelfAoe replay events are no-ops, not centered attacks. Regression anchor: Easy run `20260506_114649_974`, Mission_Teleporter turn 4 `Science_Repulse_A` stale target D4 after teleport to E3; fixed in simulator v61.

50. **Every move-then-attack target is relative to the post-move tile.** The teleporter fix must not special-case only pad moves and accidentally leave normal moves targeting from the old tile. If `auto_turn` shows `click_miss|Brute_Jetmech|attack` where Aerial Bombs was legal from the pre-move tile but not the moved tile, check `enumerate_actions`: attack target enumeration, smoke checks, and effect filtering all need the post-move attack origin. Regression anchor: Easy run `20260506_114649_974`, Mission_Teleporter turn 4 dirty recovery emitted JetMech G1→D3 then Aerial Bombs at G3, which was only legal from G1; fixed in simulator v62.

53. **ACID pools are non-stoppable for player movement.** The operational "never move onto ACID voluntarily" rule is enforced in Rust movement enumeration, not merely by scoring penalties. If a plan moves a mech onto a tile with `tile.acid=true`, treat it as a movement enumeration bug before considering weapon logic; ACID-pool status reporting can be bridge-ambiguous after the move, so the safest policy is to keep those destinations out of search for player-controlled units. Regression anchor: Easy run `20260506_114649_974`, Corporate HQ turn 2 moved RocketMech D3→C5 onto an ACID pool and produced `status|Ranged_Rocket|move`; fixed in simulator v64.

54. **Smoke on a queued web source releases the web immediately.** Scorpion/Leaper-style current-turn webs are tied to the queued attack; when a player action places smoke on that enemy, the engine cancels the queued attack and `IsGrappled()` clears before the next mech acts. The simulator must break `web_source_uid` links when smoke is placed on a live enemy tile, including Rocket Artillery's behind-shooter smoke. Regression anchor: Easy run `20260506_114649_974`, Corporate HQ turn 3 RocketMech E3→B3 smoked ScorpionBoss at F3 and freed PulseMech at F4; fixed in simulator v65.

54a. **Enemy-created smoke does not cancel later queued enemy attacks.** The
enemy attack loop latches smoke cancellation after pre-attack enemy-phase
effects and before the first queued Vek attack resolves. A Mosquito, Smoldering
Shell, or other enemy attack that smokes another queued attacker during that
loop still leaves the later already-queued attack alive; only smoke present on
that attacker's tile at loop start cancels it. Regression anchor: Bombermechs
No Survivors run `20260630_181556_177`, Detritus `Mission_Missiles` turn 3,
where a Mosquito at E3 smoked the shielded F3 Mosquito, but F3 still destroyed
the G3 building; fixed in simulator v309.

55. **Rocket Artillery damage upgrades need save overlays and Rocket semantics.** `Ranged_Rocket_A` / `Ranged_Rocket_B` are 3-damage Rocket Artillery, and `Ranged_Rocket_AB` is 4-damage Rocket Artillery. They must preserve the base weapon's smoke-behind-shooter and center corpse-push behavior. If live Rocket kills a target that sim leaves at 1 HP after a blocked push/bump, inspect save-file `state.weapons` and add the upgraded ID to both `_MODELED_UPGRADED_WEAPONS` and Rust `WId` support before advancing. Regression anchor: Easy run `20260506_114649_974`, Volcanic Hive final turn 2 `Ranged_Rocket_A` killed Scorpion2 at C6 with 3 damage plus blocked bump; fixed in simulator v66.

57. **Cannon-Bot Mark I is a projectile with fire.** `SnowtankAtk1` / Cannon 8R Mark I inherits `FireflyAtk1` in `scripts/weapons_snow.lua` and sets `Fire=1`; it is not melee. The shot travels down the queued cardinal line to the first projectile blocker and sets the hit unit on fire. If Mission_FreezeBots or Mission_Hacking shows a Cannon-Bot damaging a mech behind the first empty target tile, check this weapon definition before blaming movement or Ralph/max-HP display noise. Regression anchor: Easy Rusting Hulks run `20260508_122657_124`, Mission_FreezeBots turn 1 Cannon-Bot at G3 fired through G4/G5 into PulseMech at G6, leaving Pulse at 2 HP and on fire; fixed in simulator v67.

60. **Mission_Trapped Coal Plant tiles may need inferred unique-building accounting.** The bridge can expose 2-HP Coal Plant structures as ordinary building tiles without `unique_building=true`, but live grid power still drops when one HP is lost to push/bump damage. On `Mission_Trapped`, infer objective-style grid accounting for 2-HP building tiles before solving; do not globally change non-unique 2-HP residential bump accounting without a fresh capture. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Scrapyard turn 3 Bouncer pushed a Moth into the D4 Coal Plant tile, reducing building HP 2→1 and grid 6→5; fixed in simulator v69.

62. **Leap weapons cannot land on chasm tiles.** Aerial Bombs / leap target enumeration must reject chasm landing tiles, and replay/score diagnostics must no-op illegal chasm landings instead of damaging transit tiles. Flying does not make a Cataclysm chasm a valid Aerial Bombs landing. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Mission_Cataclysm turn 3 JetMech B5→B3 click_miss where B3 was already chasm; fixed in simulator v70.

63. **Rocket Artillery center push does not edge-bump.** When `Ranged_Rocket` hits a live center target on the map edge, the engine applies Rocket damage but does not add a blocked-push bump into the off-board edge; the target stays in place. Keep Rocket's mountain/building/blocker corpse-bump semantics from rule 42, but do not count off-board edge bump as extra kill damage. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Corporate HQ turn 2 RocketMech D7 fired at Moth1 on D1; live Moth stayed at D1 with 1/3 HP instead of dying to phantom edge bump, fixed in simulator v71.

64. **ACID status is not an instant pool under live targets.** ACID weapons such as `Acid_Tank_Attack` / A.C.I.D. Cannon acidify a live unit on the hit tile, but the ground beneath that unit does not become an ACID pool until an ACID unit dies there. Empty ground/rubble/water ACID hits can still create pools/tiles, and shielded or still-frozen targets can still use the "falls to feet" tile-acid rule when the unit cannot receive ACID. If a later move verify reports `tile_status` where `predicted acid=true` and `actual=false` on an occupied target tile, check for over-eager `apply_weapon_status` tile acid before blaming the moving mech. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Reclamation Zone turn 4 Acid Tank hit Scarab1 at E4, then JetMech D1→E5 surfaced the stale predicted pool; fixed in simulator v72.

65. **Starfish attacks are self-centered diagonal appendages, not melee.** `StarfishAtk1` / `StarfishAtk2` self-target the Starfish tile and damage the four diagonal tiles only. `StarfishAtkB1` / Starfish Leader does the same 3-damage diagonal pattern and additionally applies zero-damage outward pushes to the four cardinal adjacent tiles. If the bridge shows a Starfish queued target equal to its own tile, do not treat that as a no-op melee attack. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Corporate HQ turn 1 StarfishBoss at E3 queued `StarfishAtkB1` on itself with eight telegraphed tiles; fixed in simulator v73.

66. **Aerial Bombs cannot land on water or lava.** Even though Jet Mech is flying, `Brute_Jetmech` / Aerial Bombs uses the weapon target area for its landing tile; forced water/lava landing clicks spend the attack as a no-op and do not damage or smoke transit tiles. Reject water/lava landings in target enumeration and replay diagnostics. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Corporate HQ turn 2 JetMech D7 tried to land on B7 water over Blob1 at C7; live Jet stayed at D7 and Blob1 survived; fixed in simulator v74.

67. **Repulse edge pushes do not edge-bump.** `Science_Repulse` / Repulse is a zero-damage adjacent push. When an adjacent unit would be pushed off-board, the engine leaves it in place without adding the generic 1 bump damage. On-board Repulse pushes and blocked collisions still matter, but do not count off-board edge bump as kill damage for Repulse. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Corporate HQ turn 3 PulseMech D2 repulsed a 1-HP Blobber at D1; live Blobber stayed alive at D1 instead of dying to phantom edge bump, fixed in simulator v75.

68. **Web Egg adjacency webs are tile-based, but do not steal active grapples.** A live `WebbEgg1` webs every living unit in the four cardinal adjacent tiles, including units that move next to it after the egg already exists. Bridge `IsGrappled()` / grappler source can miss or misattribute this, so solver inputs and Rust landing/move simulation must infer adjacency and set `web_source_uid` to the egg when the unit has no live non-egg source currently targeting it. If a Scorpion/Leaper-style source is alive and its queued target is that unit's tile, preserve that source; killing the adjacent egg should not free the other grapple. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Pinnacle `Mission_FreezeBldg` turn 2 JetMech moved H2→F2 next to a F3 egg and became webbed, while Pulse remained webbed by the E2 Scorpion after the egg died; fixed in simulator v76.

69. **Burrower Slam hits the perpendicular three-tile row.** `BurrowerAtk1` / `BurrowerAtk2` are melee attacks with `path_size=3`: the queued direction selects the center target tile, and the engine also damages the two perpendicular flank tiles around that center. Do not simulate Burrowers as single-tile melee. If an objective/building loss follows a Burrower attack, first check for a missed flank tile in the slam row. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Pinnacle `Mission_Stasis` turn 1 Coal Plant at C4 was lost by a Burrower1 flank that the sim missed; fixed in simulator v77.

71. **Player artillery target areas are cardinal, not diagonal.** Standard player artillery weapons such as Artemis Artillery, Rocket Artillery, and Rock Accelerator fire along a row or column from the shooter. A bridge `FireWeapon` call can return OK on an off-axis point but spend the action as an engine no-op, so do not broaden `get_weapon_targets(... WeaponType::Artillery ...)` to board-wide targets. Regression anchors: Easy Rusting Hulks run `20260508_134925_472`, Pinnacle `Mission_Reactivation` turn 2, RocketMech E6 fired `Ranged_Rocket_A` off-axis at C4 and live Burnbug1 survived untouched; Easy Rift Walkers/Ramming Speed run `20260516_192519_866`, Detritus Nano Silos turn 1, ArtiMech D6 fired Artemis at F2 and live Crab1/Firefly2 were unchanged. Fixed in simulator v80/v134.

72. **Diagnostic replay/score must reject impossible player actions.** `score_plan` / `replay_solution` are useful only when they obey the same legality gates as live search: movement destinations must be reachable and unblocked, smoke on the attack origin prevents attacks/repair, and weapon targets must be in that weapon's legal target area. If a hand-written plan looks clean but `solve` omits it, first check for `illegal_move:*`, `illegal_attack_smoke:*`, or `illegal_weapon_target:*` events before calling it a solver miss. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Pinnacle `Mission_Reactivation` turn 2: JetMech A4→A5 then Aerial Bombs at C5 was blocked by smoke, while RocketMech E6 off-axis C4 was outside Rocket Artillery's target area; diagnostic guards hardened in simulator v80.

73. **Bridge tile arrays are records, not coordinate-indexed grids.** Recorded `bridge_state["tiles"]` can be a flat list whose order is not `x*8+y`; each tile carries authoritative `x` / `y` fields. When inspecting raw solve inputs, build a lookup from `(tile["x"], tile["y"])` instead of indexing the list directly, or you can mislabel terrain/buildings and chase a false legality bug.

75. **Normal Psions are pushable.** The shipped `pawns.lua` `Jelly_Health1` family inherits default `Pushable=true`; do not inject `pushable=false` for `Jelly_Health1`, `Jelly_Armor1`, `Jelly_Regen1`, `Jelly_Explode1`, `Jelly_Lava1`, or AE Psions. If Repulse/Rocket predictions miss bump damage through a Psion into a building, check pawn stats enrichment before changing push physics. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Archive `Mission_Repair` turn 2: PulseMech Repulse at C4 bumped Blast Psion from C5 into the 2-HP C6 building, damaging both; fixed in simulator v81.

76. **Blast Psion explosions chain through eligible non-minor Vek.** `Jelly_Explode1` grants "all other Vek explode on death"; if a Blast Psion aura burst kills another ordinary non-minor Vek, that secondary death emits its own 1-damage cardinal burst. Minor pawns still do not receive the aura, so old Totem/Spore/Blob cases remain non-chaining, and intrinsic Volatile Vek decay stays separate. Regression anchors: Easy Rusting Hulks run `20260508_134925_472`, Archive `Mission_Repair` turn 3, Totem D2 did not explode because it was Minor; Easy Rift Walkers/Ramming Speed run `20260516_120646_726`, R.S.T. Perimeter turn 2, Dash Punch killed Burnbug C4, whose burst killed Moth B4 / Leaper C3 and secondary bursts destroyed B3; corrected in simulator v125.

77. **Dam flood drown deaths still fire death side effects.** Old Earth Dam flood converts eligible tiles to water, and grounded non-massive Vek drowned by that water must go through the normal instant-death bookkeeping: kill credit, Psion cleanup, Blast Psion aura burst, and Volatile decay. Do not raw-set `hp = 0` in flood helpers. Regression anchor: Easy Rusting Hulks run `20260508_134925_472`, Archive `Mission_Dam` / Central Museums turn 2: Rocket destroyed the dam, flood drowned Shaman2 at D4, and live Blast Psion aura damaged the adjacent 2-HP Robotics Lab at D3; fixed in simulator v83.

82. **Ramming recoil on sand creates smoke.** `Brute_Beetle` / Ramming Engines self-damage is engine damage on the charger's final tile; if Charge Mech ends on sand after an impact, the sand becomes ground with smoke. Do not limit sand→smoke terrain mutation to target weapon damage only. Regression anchor: Easy Zenith Guard run `20260510_000847_773`, Mission_Terratide / Razor Bay turn 1: ChargeMech C3→E3 recoil converted E3 sand into smoke; fixed in simulator v90.

87. **Mission_BoomBots `*_Boom` pawns have Explosive Decay.** `Snowtank1_Boom`, `Snowlaser1_Boom`, and `Snowart1_Boom` use Lua `Explodes=true`: on death they deal 1 damage to the four cardinal-adjacent tiles, can kill adjacent mechs, damage/destroy buildings, and ignite forests into burning ground. Treat this as intrinsic Volatile-style decay rather than an enemy attack. Regression anchor: Easy Zenith Guard run `20260510_000847_773`, Pinnacle `Mission_BoomBots` / Central Processor turn 2: Defense Mech pulled `Snowart1_Boom` into itself, destroying the E5 building, killing Defense, damaging/burning Laser, and dropping grid after animation settled; fixed in simulator v93.

88. **Burst Beam Ally Immune needs upgraded weapon IDs.** `Prime_Lasermech_A` / `Prime_Lasermech_AB` must overlay from save-file powered mods; the Lua bridge's base SkillList only says `Prime_Lasermech`. The Ally Immune upgrade skips player-side unit damage but still lets the beam pass through the friendly tile and continue normal damage decay, while buildings/terrain remain vulnerable. If a C-Beam line appears to kill a friendly despite powered Ally Immune, check save overlay + Rust `WId` mapping before rejecting the line. Regression anchor: Easy Zenith Guard run `20260510_000847_773`, Detritus `Mission_AcidStorm` / Containment Zone D turn 4: LaserMech B3→A6 fired at B6, hitting B6/C6/E6/G6 through ScienceMech at D6 and unlocking Glittering C-Beam; fixed in simulator v94.

89. **Titan Fist Dash needs upgraded weapon IDs.** `Prime_Punchmech_A` / `_AB` must overlay from save-file powered mods; the Lua bridge's base SkillList only says `Prime_Punchmech`. Dash Punch is a `Charge`-style direction selector with no self-damage: click the adjacent cardinal direction tile, the Combat Mech charges to the last free tile, then punches the first blocker. `Prime_Punchmech_B` remains melee with 4 damage. The Lua script sets Dash cost to 2 cores and +2 Damage to 3 cores; do not assume cheaper metadata. Regression anchor: Ramming Speed setup on Easy Rift Walkers run `20260510_213059_819`, fixed in simulator v95.

93. **Mountain-destruction objectives need explicit attention until scorer coverage is proven.** Missions such as R.S.T. `Destroy 2 mountains` can be won while silently failing the mountain objective if the live solve spends all actions on kills/threats. Before ending the last two turns on these missions, check objective progress on the visible panel or solve output; if progress is still 0, prefer a clean mountain-damage line when grid/objectives remain safe, and file/fix scorer support after the run.
