## Solver Improvements Needed

### Missing Actions
- [ ] **Mech self-repair**: Not in action enumeration. Every mech can click itself to heal 1 HP + remove fire/acid. Add to `_enumerate_mech_actions` in `src/solver/solver.py`.

### Environment Awareness
- [ ] **Air strikes / tidal waves / lightning / earthquakes**: Some missions have environment hazards targeting specific tiles. Save parser doesn't extract this. Need to find where stored in saveData.lua, parse in `src/capture/save_parser.py`, add threatened tiles to eval.

### Simulation Gaps
- [ ] **Status effects incomplete**: `src/solver/simulate.py` has TODOs at lines ~220, 255, 257 for acid, freeze, shield handling.
- [ ] **evaluate_threats() hardcodes damage=1**: All enemies assumed to deal 1 damage. Needs actual weapon damage lookup from Vek data.
- [ ] **get_threatened_buildings() misses AoE/charge/splash**: Only checks exact target tile. Multi-tile attacks, charge attacks, and splash damage not accounted for.

### Execution Consistency
- [ ] **Move/attack order mismatch**: Solver simulates move-then-attack, but executor does attack-then-move. Positions may be inconsistent for weapons that depend on mech location.

### Fixed
- [x] evaluate() kill tracking via explicit `kills` parameter


## Knowledge Base Gaps

- [ ] **Vek leaders**: Stats not extracted from `data/wiki_raw/` into `data/vek.json`
- [ ] **Boss mechanics**: Barely documented in structured data
- [ ] **Advanced Edition squads**: 7 AE squads only partially covered in `data/squads.json`


## Known Bugs & Limitations

- [ ] **MECH_PORTRAIT Y-coordinates**: Only verified for Rift Walkers. Other squads have different portrait positions -- needs per-squad testing.
- [ ] **Grid coordinate calibration**: Approximate, +/-5px. Aim for lower center of tile diamond. See `grid_to_mcp()` in `src/control/executor.py`.
- [ ] **Webbed status**: Not parsed in save_parser, not modeled in Board/Unit dataclasses.
- [ ] **Between-mission phase detection**: Reward/map/shop screens not distinguishable from save file alone. Needs screenshot-based detection.
- [ ] **Save file update timing**: Unknown if saveData.lua updates per-mech-action or per-turn-end. Needs testing.
- [ ] **Profile name hardcoded**: Set to "Alpha" in save_parser. Correct for this machine but not general.
